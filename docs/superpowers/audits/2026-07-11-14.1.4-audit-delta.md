## Cleanup summary

| Change | Effect on audit |
|---|---|
| Remove `eslint` + `eslint-config-populist` + `.eslintrc.json` | Removed 15 line items from eslint subtree (all transitive CVEs that entered only via eslint) |
| Bump `tap` from `^14.11.0` to `^21.7.4` | Replaced tap 14's bundled deps with tap 21's cleaner tree; removed 11 bundled transitive CVEs plus 6 additional ones through updated dependency graph |
| Bump CI Node matrix `16.x → 20.x` | none (workflow-only, not in dep tree) |

## Explicitly not fixed in this release (deferred)

| Dep | Why deferred |
|---|---|
| `request` (dev) | Has own critical CVE (SSRF). Removal requires rewriting ~29 test-code references to `undici` or `node:fetch`. Own patch release. |
| 9 override-reachable transitive CVEs (`ajv`, `cross-spawn`, `form-data`, `mem`, `qs`, `tmp`, `tough-cookie`, `uuid`, `yargs-parser`) | Under `eslint` or `request` subtrees. `eslint` removal in this release will eliminate several automatically (`ajv`, `cross-spawn`, `tmp` — the eslint-subtree ones); those under `request` (`form-data`, `qs`, `tough-cookie`) persist until `request` itself is removed. |
| Prod-dep replacements (`corser`, `http-proxy@1`, `mime@1`, `union`, `secure-compare`) | Unrelated to current CVEs; deferred to 15.0.0 when a forcing function surfaces. |
