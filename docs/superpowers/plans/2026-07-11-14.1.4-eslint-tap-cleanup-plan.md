# `14.1.4` eslint Removal + tap Upgrade — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Publish `@pkg-nec/http-server@14.1.4` — a patch release that removes the unused `eslint`/`eslint-config-populist` dev deps, upgrades `tap@14 → tap@21.7.4`, and moves the test-CI Node matrix from `16.x` to `20.x`.

**Architecture:** Four per-topic commits (eslint removal, tap bump, CI Node bump, version bump) on branch `chore/release-14.1.4`. Between the tap-bump and the CI-Node-bump, run `npm test` locally to prove tap 21 works with the existing tests before CI changes. Then baseline + finalize an audit-delta doc, PR + CI + merge, tag + OIDC publish + verify.

**Tech Stack:** npm CLI 11+, `gh` CLI, GitHub Actions. Local test on Node 22.x (developer machine, currently blocked by `tap@14` incompatibility — this release fixes that as a side effect). Test CI moves to Node 20.x.

## Global Constraints

- **Working branch:** `chore/release-14.1.4` (already checked out). Do not commit to any other branch.
- **Merge target:** `trunk`. Branch protection requires PR + green `Test (<node-version>, ubuntu-latest)` check.
- **Target version string:** `14.1.4` — exact, verbatim, applied in the final commit only. Not sooner.
- **tap version target:** `^21.7.4` — latest published 21.x at plan-authoring time, > 7 days old (satisfies `.npmrc`'s `min-release-age=7`).
- **CI Node target:** `20.x` — per spec decision (defensible floor).
- **No code changes** under `lib/`, `bin/`, `public/`. `git diff v14.1.1-pkgnec.0 HEAD -- lib bin public` MUST remain empty at every commit.
- **No changes** to `test/*.test.js` unless tap 21's test smoke run (Task 2 Step 5) surfaces an API break — in which case the smallest possible fix lands with the tap-bump commit itself and its message flags the specific incompatibility.
- **No changes** to `.github/workflows/publish.yml`, `.github/workflows/stale.yml`, the npm Trusted Publisher config, or the `npm-publish` GitHub Environment.
- **No `engines.node` change**. Stays `">=12"` on the published package.
- **No dep replacements or prod-dep changes.** Only the eslint pair leaves; only tap bumps.
- **No `overrides` field** added to `package.json`. If one is present in the working tree from prior experiments, verify it's absent at commit tip.
- **One commit per topic**, in this exact order: (1) eslint removal, (2) tap bump, (3) CI Node bump, (4) version bump. Do NOT reorder — the CI Node bump must land AFTER local `npm test` has proven tap 21 is compatible.
- **Baseline installed values** at plan-authoring time:
  - `version` = `14.1.3`
  - `devDependencies.tap` = `^14.11.0`
  - `devDependencies.eslint` = `^4.19.1`
  - `devDependencies.eslint-config-populist` = `^4.2.0`
  - `.github/workflows/node.js.yml` matrix `node-version: [16.x]`
- **Reference points:**
  - `v14.1.1-pkgnec.0` = tag at the byte-identity baseline (used in every task's `lib/bin/public` invariant check).
  - `c8e1b67` = `trunk` at the lockfile regeneration merge (this branch's base).
  - `60ad72e` = spec commit already on this branch.
- **Baseline `npm audit`** at branch base (before any Task 1 changes): captured by Task 0 and recorded verbatim in the audit-delta doc.
- **`npm view` runs from a directory OUTSIDE this repo** (e.g., `cd /tmp && npm view …`) for post-publish verification, to bypass the local `.npmrc`'s `min-release-age=7` filter.
- **Local test signal:** local `rm -rf node_modules && npm ci && npm test` at Task 2 Step 5 is the tap-21 API-compatibility gate. If it fails with a real test error (not a flake), the tap bump commit needs to fix it before proceeding to Task 3.
