# Canary Release `14.1.1-pkgnec.1` Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Publish `@pkg-nec/http-server@14.1.1-pkgnec.1` via the OIDC workflow as a no-op smoke test, verify the trust chain end-to-end (tag → Release → environment approval → publish → provenance), then deprecate the version.

**Architecture:** Single-commit `package.json` version bump on branch `chore/canary-14.1.1-pkgnec.1` with an added `publishConfig.tag: "canary"` so the release doesn't overwrite `latest`. Merge to `trunk`, tag, create GitHub Release, approve the `npm-publish` environment gate, watch `publish.yml` fire, verify the published tarball and provenance attestation, then run `npm deprecate` to mark it as a smoke-test artifact.

**Tech Stack:** GitHub Actions, npm CLI 9+, `gh` CLI.

## Global Constraints

- **Working branch:** `chore/canary-14.1.1-pkgnec.1` (already checked out). Do not commit to any other branch.
- **Merge target:** `trunk`. Branch protection requires PR + green `Test (16.x, ubuntu-latest)` check.
- **Target version string:** `14.1.1-pkgnec.1` — exact, verbatim.
- **npm dist-tag on publish:** `canary` (via `publishConfig.tag`, NOT via a `--tag` flag on the CLI). `latest` MUST NOT move; it stays at `14.1.1-pkgnec.0`.
- **No code changes** under `lib/`, `bin/`, `public/`. Verified by `git diff v14.1.1-pkgnec.0 HEAD -- lib bin public` (must remain empty on this branch).
- **No dependency changes** — do NOT edit `dependencies`, `devDependencies`, `package-lock.json`, or `engines.node`.
- **No modifications** to `.github/workflows/publish.yml`, the npm Trusted Publisher config, or the `npm-publish` GitHub Environment. Those are the artifacts under test.
- **`files` field** in `package.json` stays `["lib", "bin", "doc"]`.
- **Reference points:** `v14.1.1-pkgnec.0` = the tag on the previous release's commit, used as the byte-identity baseline. `ef671d4` = `trunk` at PR #2 (OIDC) merge, i.e. the state of the world when this branch was cut.
- **Local `npm test` is broken pre-existing** on Node 22+ due to `tap@14` incompatibility. Test signal comes from CI (Node 16.x on ubuntu-latest, where `tap@14` works).
- **`npm view` runs from OUTSIDE this repo** for verification, to bypass the local `.npmrc`'s `min-release-age=7` filter which would hide our just-published version.

## Follow-up flagged to prevent loss

Before the `14.1.2` release publishes, `publishConfig.tag: "canary"` MUST be removed from `package.json`. If left in place, `14.1.2` would also publish under the `canary` dist-tag, and `latest` would stay stuck at `14.1.1-pkgnec.0` even after the real patch shipped. This is a one-line chore on the `14.1.2` release prep branch — do not lose it during vulnerability triage.
