# `14.1.2` Vulnerability Patch Release — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Publish `@pkg-nec/http-server@14.1.2` — a patch release that bumps dependency versions to reduce the audit surface, with one commit per dep bump for reviewability. No new logic, no breaking changes.

**Architecture:** Ten per-dep bump tasks (8 prod, 2 dev) that each run a fixed 6-step gate: check latest, decide safe target, edit `package.json` range, run `npm install`, verify no drift outside package.json/lockfile, commit. Then an audit-delta doc, a version bump commit, PR + CI + merge, tag + Release + OIDC publish + approve + verify.

**Tech Stack:** npm CLI 9+, `gh` CLI, GitHub Actions, Node.js 16.x (CI). The OIDC publish workflow (Node 22.x + npm@11.10.0) is unchanged.

## Global Constraints

- **Working branch:** `chore/release-14.1.2` (already checked out). Do not commit to any other branch.
- **Merge target:** `trunk`. Branch protection requires PR + green `Test (16.x, ubuntu-latest)` check.
- **Target version string:** `14.1.2` — exact, verbatim, applied in the FINAL commit only. Not sooner.
- **No code changes** under `lib/`, `bin/`, `public/`. `git diff v14.1.1-pkgnec.0 HEAD -- lib bin public` MUST remain empty at every commit.
- **No engine changes.** `engines.node` stays `">=12"`. `.github/workflows/node.js.yml` matrix stays `node-version: [16.x], os: [ubuntu-latest]`.
- **No dep replacements.** `corser`, `http-proxy`, `union`, `mime`, `secure-compare`, `request`, `eslint`, `eslint-config-populist`, `tap` are all out-of-scope for this release. Do NOT touch them.
- **No changes** to `.github/workflows/publish.yml`, `.github/workflows/node.js.yml`, `.github/workflows/stale.yml`, the npm Trusted Publisher config, or the `npm-publish` GitHub Environment.
- **`publishConfig`** in `package.json` at branch tip MUST have exactly `{"access": "public", "provenance": true}` — no `tag: "canary"` (removed in PR #6). Verified by `node -e "console.log('tag' in require('./package.json').publishConfig)"` returning `false`.
- **One commit per dep bump.** No batching. If a bump requires touching multiple deps together (e.g., peer-dep coordination), STOP and escalate; that's not in this plan's scope.
- **No `npm audit fix` or `npm audit fix --force`.** Every bump is a manual `npm install <dep>@<exact-or-caret-target>`.
- **Dep gate for new-major bumps:** a new major of a dep is acceptable within a patch release ONLY if ALL of the following hold:
  - Its `engines.node` still supports Node `>=12` (our declared floor, NOT our CI floor).
  - It is NOT ESM-only. Must have a CommonJS entry (`main` field, or `exports` with a CJS conditional).
  - Its changelog reports no API-surface change to what our code uses.
- **Reference points:**
  - `v14.1.1-pkgnec.0` = tag on the byte-identity baseline. Used in the `git diff … -- lib bin public` gate.
  - `9ee3bcc` = `trunk` at PR #6 merge (base for this branch). Used to scope the branch's total diff.
- **Local `npm test`** is broken pre-existing (`tap@14` on Node 22+ locally). Test signal comes from CI.
- **`npm view` runs from a directory OUTSIDE this repo** (e.g., `cd /tmp && npm view …`) for verification, to bypass the local `.npmrc`'s `min-release-age=7`.

## Per-dep bump procedure (embedded in every dep task below)

Every dep task follows this template. Each task's steps are spelled out in full (not deferred to a reference) because task briefs are extracted independently.

**Verification gates applied at Step 5 of every dep task:**

- `git diff v14.1.1-pkgnec.0 HEAD -- lib bin public` returns empty. This is the invariant that keeps the tarball's `lib/`, `bin/`, `doc/` byte-identical to upstream.
- `npm ci` after the bump succeeds with no lockfile drift.
- `git status` shows exactly two modified files: `package.json` and `package-lock.json`. Nothing else.
- `node -e "console.log('tag' in require('./package.json').publishConfig)"` prints `false` (guards against `tag: "canary"` re-appearing).

**If any gate fails**, STOP on the dep, revert with `git checkout -- package.json package-lock.json`, note the failure in the audit-delta doc, and move to the next dep.

**New-major decision (used at Step 2):** if same-major has no newer version available, check whether the next major is safe:

```bash
npm view <dep>@latest engines type main exports 2>&1
```

Accept the new major ONLY if:
- Its `engines.node` satisfies `>=12` (empty engines is fine — treat as unrestricted).
- It is NOT ESM-only. `type: "module"` combined with only ESM in `exports` (no CommonJS conditional) → reject.
- Its changelog / release notes report no API-surface change to methods we use. `grep -rn "require('<dep>')" bin lib` to see what we consume.

If ANY of these fails, stay within the current major.
