# 14.1.1-pkgnec.0 Identity Handshake Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Publish `@pkg-nec/http-server@14.1.1-pkgnec.0` — an identity-handshake release that is functionally identical to upstream `http-server@14.1.1`, so scope, package name, and publish mechanics are proven before any code-level changes ship.

**Architecture:** Ownership metadata sweep only. `lib/`, `bin/`, `public/` are untouched; the packaged tarball must be byte-for-byte identical to upstream `v14.1.1` under `lib/`, `bin/`, `doc/`. Work happens on branch `release/14.1.1-pkgnec.0`; when that branch merges to `trunk`, publish runs from a clean checkout of `trunk` at the tagged commit.

**Tech Stack:** Node.js (engines `>=12` unchanged), npm 11+, GitHub Actions, git.

## Global Constraints

- **Working branch:** `release/14.1.1-pkgnec.0` (already checked out). Do not commit to any other branch.
- **Target version string:** `14.1.1-pkgnec.0` — exact, verbatim, everywhere it appears.
- **npm package name:** `@pkg-nec/http-server` — scoped, published with `--access public`.
- **npm dist-tag on publish:** `latest` (explicit `--tag latest` flag; prereleases do not auto-move `latest` without it).
- **No code changes** under `lib/`, `bin/`, `public/`. `git diff af0ac3e HEAD -- lib bin public` must remain empty through the entire branch's life.
- **No dependency changes** — neither `dependencies` nor `devDependencies` may be edited. `package-lock.json` must stay as committed at `d6e375a`.
- **`engines.node`** stays `>=12`. Bumping it is a 15.0.0 concern.
- **`files` field** in `package.json` stays `["lib", "bin", "doc"]`. Do not add or remove entries.
- **Attribution:** the existing `contributors` array stays intact. Do not add a maintainer entry in this release.
- **Reference points:** `af0ac3e` = upstream v14.1.1 tag commit. `docs/superpowers/specs/2026-07-11-14.1.1-pkgnec.0-identity-handshake-design.md` = the source design.
- **Local test runner is known broken** on Node 22+ due to `tap@14` (devDependency, 2021) incompatibility with modern Node. Verified: fails identically at `af0ac3e` (upstream v14.1.1) with `Cannot find module './reports/terse'`. Pre-existing; not introduced by this release. Upgrading `tap` belongs in `14.1.2`'s vulnerability sweep, not here. Test signal for this release comes from CI (Node 12–17 matrix on GitHub Actions, old enough that `tap@14` works) plus the tarball-identity check (`git diff af0ac3e HEAD -- lib bin public` empty) which proves no code regression relative to upstream.
